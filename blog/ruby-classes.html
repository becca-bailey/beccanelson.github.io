Classes, Objects, and Kitties

Object-Oriented Ruby: Cat Edition

October 17, 2015

While learning Ruby, one of the hardest concepts for me to grasp has been object-oriented programming. At first it seemed pretty unnecessary, and it made more sense just to keep creating regular old local variables and telling the computer what to do.

But, in an object-oriented language like Ruby or JavaScript, that makes about as much sense as being this cat, trying to program a computer. So instead, I'm going to program this cat.

First, I need to create a class.

class Cat

This makes 'Cat' an object (or in this case, a warm-blooded quadriped). We're going to give this cat a name, a breed, and a status.


  attr_reader :name, :breed, :hungry?, :mood
  def Initialize(name, breed)
    @name = name 
    @breed = breed 
    @hungry? = true
    @mood = 'unhappy' 
  end

In English, this means that we can access the cat's name, breed and status outside of the class. Also, they are now instance variables, meaning they can be accessed in other methods within the class. I can now create a new cat.


  gilbert = Cat.new('Gilbert', 'American Shorthair')

  gilbert.name
  => "Gilbert" 

  gilbert.breed 
  => "American Shorthair"
  

            
Note that if I wanted to change Gilbert's name to Phil, I couldn't do that unless I changed :name to attr_accessor.  I don't really want to do that, though.  Now, I have a hungry and unhappy cat.  How can I fix that, I wonder?  Maybe I should write a method that goes inside the Cat class.
 

            def feed 

              @hungry? = false 

            end 


            def pet 

              @mood = 'happy' 

            end

            
Now, I can feed and pet my cat by calling these methods outside of the class.


            gilbert.feed 

            gilbert.pet 
 

            gilbert.hungry? 

            => false 
 

            gilbert.mood 

            => "happy"

            
See, now I have a happy kitty.  Isn't it amazing what classes can do?  If only real pets were this easily pleased.
