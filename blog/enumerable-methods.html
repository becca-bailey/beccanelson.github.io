Map and Collect

Two Methods That Do the Same Thing

October 11, 2015

If you're new to Ruby, you may have noticed there are a lot of different methods you can use, particulary when working with arrays. Here's a new one I discovered this week.

Let's say you have an array of numbers, and you need to add ten to each one.

numbers=[26, 3, 90, 45, 15]

You could use the each method to print them out, like this.

numbers.each do |n|
  n += 10
  p n + " "
end
=> 36 13 100 55 25

But what if you still wanted an array? This is where you can use the map method to apply a code block to each item in an array.

numbers.map { |n| n += 10 }
=> [36, 13, 100, 55, 25]

But, now look what happens when we print the array.

p numbers
=> [26, 3, 90, 45, 15]

Ruby will return the original array, becuase map is by default a non-destructive method. If you want to make a method destructive (or as we like to say—"dangerous"), you can add a !, also called a bang. This sybol doesn't just mean you're doing something enthusiastically—it also means you are changing the original array. Now let's see what happens.

numbers.map! { |n| n += 10 }
=> [36, 13, 100, 55, 25]
p numbers
=> [36, 13, 100, 55, 25]

Now, I have one more trick to show you.

numbers.collect! { |n| n += 10 }
=> [36, 13, 100, 55, 25]

See what I did there? The collect method does the same thing as map. "What's the difference?" you might ask. My brilliant answer: there is none. They are exactly the same. Why, Ruby.... whyyyyy??

Did you now that before? Well, you do now.
