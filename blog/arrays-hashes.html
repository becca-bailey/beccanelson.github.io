Using Arrays and Hashes

October 4, 2015

In Ruby (and other programming languages), arrays and hashes are ways of organizing data. An array is simply a container for holding data. Allow me to demonstrate this concept, and show off my stellar household organizational skills in the process.

Arrays

This is a box full of stuff. You can tell how organized I am at the moment, because in this box I have about 27 balls of yarn, a pair of socks, a box of colored pencils, a bag of trail mix, and the movie “500 Days of Summer.” I could turn this box into an array like this.

box = [“yarn”, “socks”, “colored pencils”, “trail mix, “500 Days of Summer”]

Note that the items in my array, like the items in my house, are in no particular order. Also, an array can contain strings of text (like in my example), numbers, booleans (true or false statements), and even other arrays.

Each item in the array has a number, starting with zero. For example, if I told my program to return box[0], it would return the first item, which is "yarn". I could also count backwards, and access the last item using the number -1.

Hashes

Now, let's say I was an organized person who labeled every item by type. Instead of a random list, I would have something more like this.

box = { "material" => "yarn", "clothing" => "socks", "art supply" => "colored pencils", "snack" => "trail mix", "movie" => "500 Days of Summer" }

This would be a hash. Hashes are great for storing information that has a clear identifier, such as "name" => "Becca". This is called a key, and a value. Instead of using a number to access data like you would for an array, you would access information using its key. For example, return box["snack"] would return "trail mix"

There is a lot more information to be found about constructing arrays and hashes, adding and removing data, and using Ruby methods to iterate through them, so I have inclded some more resources here.
