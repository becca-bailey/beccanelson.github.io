If you've been learning tech skills for a while now, I'm sure you've heard a lot about Ruby on Rails and how it can be used to build web apps. But what is Rails, exactly?

If you're not sure, this is a good place to start.

Here's the basic idea:

Ruby is a great programming language on its own, but needs a little help to program the web. Unlike CSS, HTML, and JavaScript which are built into all web browsers, in order to work on the web, Ruby needs a framework, the most common being Rails. Rails has been used to build popular sites like Air BnB, Dribble, and even GitHub.

To learn a little more about Rails, I went through the Getting Started with Rails tutorial at RailsGuides.

While I'm not going to take you through the whole tutorial, here are some interesting bits of information I learned in the process.

  1. Though Rails can help to build the back-end of a mobile app, Rails itself doesn't build mobile apps for iOS and
  2. Rails is a package library (more specically a Ruby Gem) that can be installed and operated using the command line.
  3. Rails is said to be "opinionated", meaning it is less flexible than Ruby alone, and requires the developer to learn the "right" way to program.

So basically, Rails is a HUGE topic that would take a lot of time to fully explore, and I'm sure there will be plenty of time for that in Phases 1, 2, and 3 of Dev Bootcamp. But for now, here's some more resources to get started.
